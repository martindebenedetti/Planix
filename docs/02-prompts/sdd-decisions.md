# Decisiones SDD (Spec-Driven Development)

Para el proyecto se decidió aplicar la metodología Spec-Driven Development.

## Estructura de specs

Cada rol debe definir su especificación técnica antes de comenzar a desarrollar.

Archivos:

- spec-devops.md
- spec-frontend.md
- spec-ux.md
- spec-ia.md

## Contenido de cada spec

Cada spec debe incluir:

- Contexto del problema
- Objetivo de la tarea
- Alcance
- Criterios de aceptación

## Validación

Cada Pull Request deberá incluir su archivo spec correspondiente.  
Un PR sin spec asociada no será aprobado.
